1. Для чого призначений Collections Framework? 2Java Collections Framework — це єдина архітектура для представлення та маніпулювання колекціями (групами об'єктів). Вона призначена для:Зберігання даних: Надання готових структур даних (списки, множини, черги, карти).Маніпулювання даними: Забезпечення стандартних алгоритмів для пошуку, сортування, заміни, перемішування даних тощо.Підвищення продуктивності: Високоякісні реалізації структур даних звільняють програміста від написання власного низькорівневого коду.2. В чому полягає різниця між масивами та колекціями? 3Розмір: Масиви мають фіксований розмір, який задається при створенні. Колекції динамічні — вони можуть автоматично збільшуватися або зменшуватися при додаванні/видаленні елементів.Тип елементів: Масиви можуть зберігати як примітивні типи (int, double), так і об'єкти. Колекції можуть зберігати тільки об'єкти (для примітивів використовуються класи-обгортки, наприклад Integer замість int).Функціональність: Колекції надають багатий набір методів (додавання, видалення, перевірка наявності, очищення), тоді як масиви мають обмежений функціонал.3. Які особливі властивості є у списків (List) порівняно з іншими типами колекцій? 4Основна особливість списків (List):Впорядкованість: Елементи зберігаються у суворій послідовності.Доступ за індексом: Кожен елемент має свій числовий індекс (починаючи з 0), що дозволяє точково звертатися до будь-якого елемента (get(int index)), як у масивах.Дублікати: На відміну від множин (Set), списки дозволяють зберігати однакові елементи (дублікати).4. Чим списки відрізняються від масивів? 5Хоча списки (особливо ArrayList) схожі на масиви логікою доступу за індексом, вони відрізняються:Динамічність: У список можна нескінченно додавати елементи (поки вистачає пам'яті), масив же переповниться.Методи: Список має вбудовані методи для видалення елемента зі зсувом решти елементів (remove), вставки всередину (add(index, element)), чого масив не вміє робити автоматично.Вивід: При виведенні списку на екран (System.out.println(list)) ви побачите вміст, а при виведенні масиву — часто лише хеш-код (без використання Arrays.toString()).5. Чим ArrayList відрізняється від LinkedList? 6ArrayList: Базується на динамічному масиві.Плюси: Дуже швидкий доступ до елемента за індексом ($O(1)$).Мінуси: Повільна вставка та видалення елементів в середину списку (потребує зсуву масиву).Коли використовувати: Коли потрібно часто читати дані (get), але рідко змінювати список.LinkedList: Базується на двозв'язному списку (кожен елемент зберігає посилання на попередній та наступний).Плюси: Швидка вставка та видалення елементів ($O(1)$), якщо є посилання на позицію.Мінуси: Повільний доступ за індексом ($O(n)$), бо треба перебирати елементи по черзі.Коли використовувати: Коли часто додаються або видаляються елементи (особливо на початку або в середині), або при реалізації черг/стеків.6. Що таке Iterator? Переваги над циклом for(int i...). 7Iterator — це об'єкт, який дозволяє послідовно перебирати елементи колекції.Переваги:Універсальність: Ітератор працює для всіх колекцій (Set, List, Queue), тоді як цикл for(int i...) працює тільки для тих, що мають індекси (List).Безпечне видалення: Метод iterator.remove() дозволяє безпечно видаляти елементи під час перебору. У звичайному циклі for видалення елемента може призвести до помилок (пропуску елементів або ConcurrentModificationException).Абстракція: Приховує внутрішню структуру колекції від програміста.7. Що таке типізовані та нетипізовані колекції? 8Нетипізовані (Raw types): Використовувались до Java 5. Зберігають об'єкти типу Object. При отриманні елемента потрібно вручну робити приведення типу (casting). Це небезпечно, бо помилку типу можна помітити лише під час виконання програми.Приклад: ArrayList list = new ArrayList();Типізовані (Generics): Вказують конкретний тип даних, що зберігатиметься. Компілятор перевіряє типи автоматично, приведення типів не потрібне.Приклад: ArrayList<Student> list = new ArrayList<>();8. Для чого потрібен інтерфейс RandomAccess? 9Це інтерфейс-маркер (не містить методів). Він використовується, щоб повідомити алгоритмам, що дана колекція (наприклад, ArrayList) підтримує швидкий (довільний) доступ до елементів за індексом за константний час.Алгоритми (наприклад, Collections.binarySearch) перевіряють цей маркер: якщо він є, використовують доступ за індексом; якщо немає (як у LinkedList), використовують ітератор для перебору, щоб не втрачати продуктивність.9. Чим Collection відрізняється від Collections? 10Collection (без "s") — це інтерфейс. Це корінь ієрархії колекцій (від нього походять List, Set, Queue), який визначає загальні методи (add, remove, size).Collections (з "s") — це утилітний клас. Він містить лише статичні методи для роботи з колекціями, такі як Collections.sort(), Collections.reverse(), Collections.max() тощо.
