1. Для чого призначений Collections Framework? 
Collections Framework — це уніфікована архітектура для представлення та роботи з колекціями (групами об'єктів).
Вона призначена для:
Зберігання даних за допомогою готових структур (списки, множини, черги, карти).
Маніпулювання даними через стандартні алгоритми (сортування, пошук, заміна).
Зменшення зусиль на програмування та підвищення продуктивності завдяки високоефективним реалізаціям алгоритмів і структур даних.

2. В чому полягає різниця між масивами та колекціями? 
Розмір: Масиви мають фіксований розмір, заданий при створенні.
Колекції — динамічні, їх розмір змінюється автоматично при додаванні чи видаленні елементів.
Типи даних: Масиви можуть зберігати як примітивні типи, так і об'єкти.
Колекції працюють лише з об'єктами (примітиви загортаються в класи-обгортки, наприклад Integer).
Функціонал: Колекції надають широкий набір готових методів для роботи з даними, тоді як масиви мають обмежений функціонал.

3. Які особливі властивості є у списків порівняно з іншими типами колекцій? 
Списки (List) — це впорядковані колекції, які допускають дублювання елементів. Їхня ключова властивість — це доступ до елементів за числовим індексом (позицією).
Це дозволяє точно контролювати місце вставки елемента та отримувати доступ до конкретного елемента так само, як у масивах.

4. Чим списки відрізняються від масивів? 
Списки можуть динамічно змінювати свій розмір (розширюватися), а масиви — ні.Списки підтримують вставку елемента всередину зі зміщенням решти елементів та автоматичне видалення ("схлопування" прогалин), чого масиви не роблять самостійно.
Списки мають зручне рядкове представлення при виведенні на друк, на відміну від стандартного виведення масивів (яке показує хеш-код).

5. Чим ArrayList відрізняється від LinkedList? Який з них у яких випадках краще використовувати? 
ArrayList базується на динамічному масиві. Забезпечує миттєвий доступ до елементів за індексом (O(1)), але повільну вставку/видалення в середині списку (через зсув елементів).Коли використовувати: Для зберігання даних, які часто зчитуються, але рідко змінюються.LinkedList базується на двозв'язному списку. Забезпечує швидку вставку та видалення елементів (O(1)), але повільний доступ за індексом (O(n)).Коли використовувати: Коли в програмі часто відбуваються операції додавання чи видалення елементів (особливо на початку або в середині колекції).

6. Що таке Iterator? В чому полягають переваги використання ітератора у порівнянні з класичним циклом? 
Iterator — це об'єкт, який дозволяє послідовно перебирати елементи колекції. Переваги над циклом for(int i...):Дозволяє безпечно видаляти елементи під час перебору (iterator.remove()), уникаючи помилок ConcurrentModificationException.Є універсальним для всіх колекцій, включаючи ті, що не мають індексів (наприклад, Set).Приховує внутрішню реалізацію колекції, дотримуючись принципу інкапсуляції.

7. Що таке типізовані та нетипізовані колекції? 
Нетипізовані (Raw types): Зберігають елементи як тип Object. Потребують явного приведення типів (casting) при отриманні даних, що небезпечно через можливі помилки типу під час виконання (ClassCastException).Типізовані (Generics): Використовують узагальнення (наприклад, List<String>) для вказівки конкретного типу елементів. Компілятор автоматично перевіряє відповідність типів, що робить код безпечнішим і усуває потребу в ручному приведенні типів.

8. Для чого потрібен інтерфейс RandomAccess? 
RandomAccess — це інтерфейс-маркер (не містить методів), який вказує на те, що дана реалізація списку (наприклад, ArrayList) підтримує швидкий (довільний) доступ до елементів за індексом за константний час. Алгоритми (наприклад, бінарний пошук Collections.binarySearch) використовують цю мітку для вибору найбільш ефективної стратегії обробки даних.

9. Чим Collection відрізняється від Collections? 
Collection — це кореневий інтерфейс ієрархії колекцій, який визначає базові методи для роботи з групами об'єктів (додавання, видалення, перевірка розміру).Collections — це допоміжний (утилітний) клас, який складається виключно зі статичних методів для обробки колекцій (наприклад, sort, reverse, shuffle, max).
